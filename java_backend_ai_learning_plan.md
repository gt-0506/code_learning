# Java 后端 + AI 时代学习路线与实战计划

这份路线的目标不是把所有视频完整刷完，而是用最短路径恢复编码能力、建立后端系统思维，并做出能写进简历和用于面试表达的项目。

核心原则：

- 视频是地图，项目是地面。
- 语法不用死背，但必须上手。
- AI 可以帮你生成代码，但架构判断、调试、测试和复盘要自己完成。
- 每个阶段都要产出一个小项目或把已有项目推进一层。
- 每学一个技术，都要能回答：它解决什么问题、适合什么场景、容易出什么 bug、我在项目里怎么用了。

推荐主线：

```text
Java基础 -> Git/Maven -> MySQL -> Spring Boot -> Redis -> Docker
-> RabbitMQ/RocketMQ -> Spring高级 -> JVM -> Spring Cloud/微服务
-> 综合项目 -> 面试复盘 -> 黑客松尝试
```

---

## 总学习策略

每个模块按下面节奏学习：

1. 快速过一遍目录，知道这门课覆盖什么。
2. 核心章节认真看，暂停、敲代码、跑通。
3. 非核心章节快速过，有需要时回查。
4. 用 AI 辅助生成重复代码，比如实体类、Controller、Mapper、测试脚本、Docker Compose。
5. 自己负责关键判断，比如事务边界、索引设计、缓存策略、MQ 幂等、异常处理。
6. 每阶段做一个小项目。
7. 写 README，总结架构、测试方式、遇到的问题和解决方案。

视频分三类看：

| 类型 | 学法 |
|---|---|
| 核心能力 | 慢看，跟着敲，必须跑通 |
| 工具配置 | 快看，需要时回查 |
| 八股/源码/扩展 | 先扫一遍，项目遇到问题再深入 |

每天建议节奏：

- 1.5 小时看重点视频
- 2 小时跟着写或自己改
- 1 小时做阶段项目
- 30 分钟写笔记和 README

笔记格式建议：

```text
知识点：
解决问题：
使用场景：
常见 bug：
我在项目里怎么用了：
```

## 综合项目方向

完成基础阶段后，建议做一个更像现代企业项目的综合项目。

项目：AI + 企业知识库工单助手

要求：

- 用户上传文档
- 文档解析、切片、入库
- 用户提问，返回答案和引用来源
- 工单自动分类
- 管理端查看高频问题
- Redis 缓存热点问答
- MQ 异步处理文档解析
- Docker Compose 部署
- 可选：接 OpenAI 或本地模型 API

测试方式：

## 黑客松预备方向

如果想尝试 Web3 黑客松，可以从 Java 后端擅长的方向切入，不要一开始硬冲底层区块链。

项目：链上活动积分平台

要求：

- 用户连接钱包
- 后端绑定钱包地址和用户
- 监听某个测试网合约事件
- 根据链上行为给用户发积分
- MySQL 记录积分流水
- Redis 做排行榜
- MQ 异步处理链上事件
- 前端展示用户积分和活动记录

测试方式：

- 使用测试网，不碰真钱
- 模拟重复链上事件，积分不能重复发
- 模拟事件监听服务中断，恢复后能补偿扫描
- 验证钱包地址签名，不能随便冒充别人
- 录制 demo：连接钱包 -> 完成链上动作 -> 后端监听 -> 积分变化

这个项目适合作为 Web3 黑客松入门，因为它把 Java 后端、MySQL、Redis、MQ 和区块链事件结合起来。

## 学完后你应该能说清楚什么

项目完成后，面试或简历里不要只写“熟悉 Spring Boot、Redis、MQ”。

更好的表达是：

```text
我做过一个可部署、可压测、能处理并发、消息重试、缓存一致性和服务降级的系统。
开发过程中使用 AI 辅助生成代码和测试脚本，但核心架构设计、问题定位和技术取舍由我完成。
```

每个项目 README 至少包含：

- 项目背景
- 技术栈
- 架构图或模块说明
- 核心业务流程
- 数据库设计
- Redis 使用场景
- MQ 消息流转
- Docker 启动方式
- 测试方式
- 遇到的问题和解决方案
- 后续可优化点
